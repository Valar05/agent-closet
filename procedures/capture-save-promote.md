# Capture, Save, Promote

Type: Procedure
Status: Active
Date: 2026-06-22
Related doctrine: Location-Aware Continuity, Shared Brain Governance, Crucible Protocol

## Purpose

Prevent useful discoveries from dying in chat.

## Rule

A discovery is not captured until it is:

1. Named.
2. Stored.
3. Indexed.
4. Retrievable.
5. Verified by readback.

## Procedure

### 1. Observe

Read the relevant artifacts first.

Do not advise from vibes when files, docs, repos, logs, or registries are available.

### 2. Classify

Choose the asset type:

- Discovery: raw useful realization.
- Pattern: recurring structure across examples.
- Procedure: repeatable workflow.
- Judgment: decision rule or doctrine.
- Automation Candidate: work that should eventually run without prompting.
- Agent Profile: role behavior and inheritance.
- Glue Layer: missing interface between systems.

### 3. Capture Required Fields

Every durable asset should include:

- Title
- Type
- Date
- Status
- Summary
- Source / context
- Retrieval keywords
- Related assets
- Reuse instructions
- Acceptance criteria

### 4. Promote

Use Crucible tests:

- Repetition: does it recur?
- Utility: does it reduce future work?
- Compression: does it explain multiple smaller things?
- Transfer: does it apply across domains?
- Inheritance: should future agents act differently because of it?

### 5. Save

Store in the correct repo/folder.

For Agent Closet:

- shared doctrine -> `shared/`
- agent roles -> `agents/<name>/`
- procedures -> `procedures/`
- glue layers -> `glue/`
- rough notes -> `scratch/`

### 6. Readback

Fetch or reopen the saved artifact.

If it cannot be read, it is not saved.

### 7. Index

Add links from the relevant registry or manifest.

## Red Build Pattern

Claiming capture without readback is not capture.

## Acceptance Criteria

This procedure passes when a future agent can find and apply the saved discovery without asking Drew to reconstruct it.
