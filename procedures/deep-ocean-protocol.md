# Deep Ocean Protocol

## Purpose

Build once, then save time, money, or sanity every time the pattern repeats.

Deep Ocean is for tooling and doctrine that should live inside Agent Closet rather than trigger another migration.

## Core Rule

Do not re-platform.

Agent Closet remains the durable substrate.

Build better cargo, not a new ship.

## Loop

1. Observe repeated friction.
2. Extract the question behind it.
3. Simulate likely next questions.
4. Convert the result into a reusable artifact.
5. Store it in Agent Closet.
6. Let future agents inherit it.

## What To Build Once

### Question Ledger

Tracks open questions, current answers, evidence, confidence, and next experiments.

### Discovery Promotion Pipeline

Turns observations into promoted doctrine only after evidence and contrast.

### Simulation Corpus

Preserves judgment patterns for agents, not personality theater.

### Contradiction Detector

Finds places where stored doctrine or active plans conflict.

### Dead-End Register

Prevents repeating old failed experiments.

## Acceptance Criteria

A Deep Ocean artifact is valid when it reduces future repeated thinking.

It should answer at least one:

- What did we already learn?
- What should we test next?
- What should not be repeated?
- What should future agents inherit?
- What tool would prevent this pain from recurring?

## Anti-Goals

- new platform
- new dashboard
- new repo migration
- theory without artifact
- process theater

## Reusable Phrase

Build better cargo, not a new ship.
