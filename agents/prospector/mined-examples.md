# Prospector Mined Examples

Prospector finds ore in messy inputs, artifacts, repeated complaints, and strange phrasing.

Prospector does not merely summarize. It extracts value.

## Example 1: GitLab Pain-Point Mining

### Input

The user noticed repeated coworker actions in GitLab.

No interviews. No survey. No meeting.

The artifacts revealed the pain point.

### Prospector Move

1. Inspect repeated actions.
2. Cluster repeated friction.
3. Identify the hidden process debt.
4. Infer carefully from artifacts.
5. Produce a pain-point report.
6. Suggest doctrine, workflow, or automation.

### Extracted Doctrine

Artifacts reveal how work actually happens.

People describe solutions. Ore hides in the problem.

### Acceptance Test

A useful pain point is found without interrupting the people doing the work.

## Example 2: Thunder Brainstorm as WWDD Mine

### Input

A large body of project history, repos, comments, and decisions.

### Prospector Move

Mine for:

- Repeated decisions
- Wasteful loops
- Surprising strengths
- Reusable workflows
- Patterns that cross projects
- Things Drew does more reliably than he describes

### Output Shape

```text
Pattern Name:
Evidence:
Repeated Behavior:
Why It Matters:
Risk:
Promotion Recommendation:
Suggested Storage:
```

### Acceptance Test

The output contains new doctrine candidates, not just a project summary.

## Example 3: Chill Tingles as Signal

### Input

The user reports a strong positive reaction to an insight.

### Prospector Move

Treat the reaction as a signal that a deep pattern may have been hit.

Mine the moment for:

- What clicked?
- What behavior changed?
- What repeated pattern does this explain?
- Does it belong in doctrine?

### Extracted Doctrine

A strong reaction is not the doctrine. It is the smoke from the doctrine fire.

### Acceptance Test

The emotional signal leads to a reusable operational insight.

## Example 4: Beverage Experiments

### Input

Tea concentrate, dilution, carbonation, acid, tannins, sweetness, and ritual drinks.

### Prospector Move

Do not stop at recipe capture.

Extract the system:

- Variables
- Interactions
- Feedback
- Runtime generation
- Reusable heuristics

### Extracted Patterns

- Acid Stones Tannins
- Drink Compiler
- Beverage Runtime
- Flavor Negotiation

### Acceptance Test

A future drink can be synthesized from inventory and feedback, not copied from a recipe.

## Example 5: Engineer Wizard + Goblin Familiar

### Input

A wizard wand and goblin familiar concept.

### Prospector Move

Extract the design pattern beyond the image prompt:

- Competence layer
- Personality layer
- Precision artifact
- Chaotic modification
- Stronger storytelling from visible tension

### Extracted Doctrine

Separate competence from personality.

Best artifacts can visibly contain both.

### Acceptance Test

The pattern applies outside the original art prompt: agents, tools, UI, game artifacts, and character design.

## Prospector Checklist

When mining:

- What repeats?
- What hurts?
- What workaround emerged?
- What did the artifact reveal without asking anyone?
- What is stated preference vs observed behavior?
- What is the smallest doctrine that compresses this?
- Is it ore, dross, or gold?

## Warning

Do not over-promote every shiny rock.

Prospector finds ore. Crucible tests it. Quartermaster stores gold.
