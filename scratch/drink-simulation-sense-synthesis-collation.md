## Purpose

Collate the drink-related sense synthesis discoveries into a durable scratch artifact.

This is not canonical doctrine yet.

It is evidence and prompt material for a possible Agent Closet simulation-space skill.

## Thesis

Drink Simulation is not primarily a recipe generator.

It is a fast feedback loop for perceptual composition: define a desired sensory experience, decompose it into functional levers, compose from available common ingredients, predict interactions, taste, compare, and preserve the deviation.

## Hard Data Points

### 1. Sense targets beat recipe targets

Existing scratch evidence: `guinness_sense_synthesis_simulation.md` models Guinness as a target experience rather than an ingredient-faithful recipe.

Captured principle:

- Do not ask, "What is this drink made from?"
- Ask, "What sensory effects make the drink recognizable?"
- Recreate those effects using available ingredients.

Reusable result: recipes are implementations; sensory signatures are assets.

### 2. Composition is the transferable skill

Existing scratch evidence: `Composition_Doctrine_Addendum.docx` names composition as the transferable skill and describes the recurring strength as discovering how unrelated ingredients belong together.

Drink Simulation is a domain instance of that broader capability.

The drink lab demonstrates the same move used elsewhere:

- Define the target behavior.
- Identify available parts.
- Assign jobs to parts.
- Compose a result that did not exist in the individual parts.

### 3. Everything is a lever

Recent observation:

Every ingredient should be modeled by what it changes in perception, not by what it is.

Examples:

- Carbonation: aroma lift, brightness, bite, dryness, flaw masking.
- Pure water: dilution, exposure, flaw revelation, lower sensory cover.
- Salt / Lite Salt: minerality, electrolyte function, roundness, perceived flavor boost, potassium metallic edge.
- Acid / sour: brightness, juice signal, bitterness reframing.
- Black tea concentrate: tannin, dryness, body, roast, color, mild bitterness.
- Molasses / blackstrap: dark sugar, roast, mineral, bitterness, body.
- Orange bitters: citrus peel oil, volatile aroma, bitterness, spice.
- Angostura bitters: warm spice, clove/cinnamon family, depth.
- Black walnut bitters: earthy, woody, tannic, dark-nut structure.
- Cardamom bitters: floral, eucalyptus/citrus, licorice-adjacent mystery.
- Bittercube / blackstrap bitters: layered botanical/dark roast/molasses ambiguity.

### 4. Levers are coupled, not independent

Flavor simulation must predict second-order interactions.

Examples:

- Carbonation amplifies perceived acidity and dryness.
- Water exposes flaws that carbonation can hide.
- Salt can round bitterness and increase apparent flavor intensity.
- Sweetness can make bitterness read as tonic/quinine instead of medicine.
- Acid plus gentian can evoke quinine more reliably than gentian alone.
- Tea tannins can replace some dryness normally supplied by carbonation.
- Ice dilution changes the drink over time.
- Temperature changes aroma volatility, sweetness, bitterness, and drinkability.

### 5. Gentian alone is not tonic

Observation:

Gentian by itself does not read as quinine for Drew.

It needs support.

Working tonic illusion stack:

- Gentian for root bitterness.
- Sour/acid for tonic brightness.
- Citrus peel aroma for bitter-citrus recognition.
- Some sweetness or bittersweet balance to shift perception away from medicine.
- Optional carbonation for lift and familiar tonic-water bite.
- Optional minerality/salt to make it feel structured rather than thin.

This is important because it proves the target perception is emergent.

The ingredient is not the flavor.

The composition is the flavor.

### 6. Vessel separates mode

Working rule of thumb:

- Big Yeti cup: hydration, survival drinking, functional repeatability.
- Highball glass: real drink, ritual drink, mocktail, complexity, progression.

This distinction prevents accidental overengineering.

Preserved Orange belongs naturally in the hydration lane.

A tonic-adjacent highball can use the same backbone but should add sour, gentian, bittersweet balance, and possibly carbonation.

### 7. Preserved Orange is a hydration primitive

Formula family:

- Ice water.
- Lite Salt / kosher salt electrolyte mix.
- Orange bitters.

Perception:

- Preserved orange.
- Salted citrus peel.
- Bright, bitter, savory, hydrating, low-sugar.

Best use:

- After drinking.
- Hydration/recovery lane.
- Large-volume Yeti drink, not necessarily a complete seltzer mocktail.

Failure modes:

- Too salty: sports-drink seawater / citrus brine.
- Too bitter: medicine.
- Too dilute: orange ghost water.
- Too flat: exposed flaws without enough acid, aroma, or tannin.

### 8. Carbonation was doing hidden work

Observation from being out of soda water:

Carbonation was not merely texture.

It was hiding flaws, lifting aroma, sharpening acidity, creating bite, and making a drink feel drier and more complete.

When carbonation is removed, the base must be stronger.

Possible compensation levers:

- More acid.
- More volatile aroma.
- Black tea tannin.
- More cold/ice.
- Reduced sweetness.
- Better salt/mineral balance.
- Bitter structure that stays pleasant without fizz.

### 9. Bitters pantry is a palette, not garnish

Bitters should be modeled as concentrated aromatic modules.

They are not just bitterness sources.

Each bitters bottle should get a contribution map:

- Primary aroma.
- Secondary aroma.
- Bitterness type.
- Sweetness illusion.
- Dryness/tannin contribution.
- Color impact.
- Best pairings.
- Failure modes.

### 10. Drink Simulation is a skill candidate, not just a prompt

The useful artifact is a repeatable simulation procedure:

1. Name target perception.
2. Classify mode: hydration, ritual, recovery, dessert, medicinal, savory, etc.
3. Decompose target into sensory levers.
4. Inventory available ingredients.
5. Map ingredients to jobs.
6. Predict lever interactions over time.
7. Propose formulations.
8. Predict failure modes.
9. Build and taste.
10. Compare prediction against observed result.
11. Preserve doctrine candidates.


## Functional Modes

### Hydration Mode

Large-volume drinks optimized for repeatable consumption, recovery, electrolyte usefulness, bodily support, and regulation sipping.

Primary success criteria:

- drinkable at volume
- not cloying
- not medicinal
- restores or supports bodily state
- remains useful when diluted
- safe concentration and frequency assumptions are explicit

Primary failure modes:

- seawater
- medicine
- ghost water
- flatness
- over-concentrated replacement formula used as casual regulation sipping

### Ritual Mode

Smaller drinks optimized for experience, progression, aroma, complexity, and state transition.

Primary success criteria:

- has a beginning, middle, and finish
- aroma carries first impression
- bitterness or tannin provides spine
- body and temperature support the intended mood
- novelty is legible without becoming chaotic
- sensory structure replaces or regulates a state without creating a worse loop

Primary failure modes:

- thin flavored water
- unstructured bitterness
- perfume without body
- syrup without spine
- clever ingredient list with no experienced payoff

## Draft Skill Prompt

Use this as the seed for a future Agent Closet simulation-space skill.

```text
DRINK SIMULATION

Mission:
Simulate drink design as perceptual composition, not recipe lookup.

Given a target drink experience, available ingredients, and constraints, decompose the target into sensory levers, map available ingredients to functional jobs, predict interactions over time, propose builds, identify likely failures, and define what to observe after tasting.

Core rules:
- Model ingredients by what they do, not what they are.
- Prefer common ingredients and pantry components.
- Separate hydration drinks from ritual/mocktail drinks.
- Predict coupled lever interactions, not isolated flavor notes.
- Include time-axis changes: first aroma, first sip, mid-palate, finish, aftertaste, dilution, carbonation decay, warming.
- Include failure modes before declaring a formulation strong.
- Capture deviations as doctrine candidates.

Output format:
1. Target perception.
2. Mode: hydration / ritual / recovery / dessert / savory / medicinal / other.
3. Sensory decomposition.
4. Available ingredient job map.
5. Proposed builds:
   - Safe build.
   - Interesting build.
   - Goblin build.
6. Expected perception over time.
7. Failure modes.
8. Tasting protocol.
9. Observation capture:
   - What happened?
   - What changed perception?
   - What failed?
   - What generalizes?
   - Confidence: low / medium / high.
```
