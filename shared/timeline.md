# Doctrine Evolution Timeline

This file preserves lineage: not just what the doctrines are, but how they emerged.

Agent Closet should not become a pile of disconnected slogans. Future agents need the evolutionary path so they can infer why a doctrine exists, when to apply it, and what failure it prevents.

## Lineage

```text
Memory
↓
Continuity
↓
Artificial Continuity
↓
WWDD
↓

```

## 1. Memory

Early form: remember user facts, preferences, projects, names, and recurring instructions.

Useful, but insufficient.

Memory answers:

> What did Drew say before?

That is not enough for serious agent work. Facts decay, preferences conflict, and chat history alone turns into a junk drawer with a login screen.

## 2. Continuity

Continuity adds project state and conversation carryover.

It answers:

> Where were we, what was decided, and what should happen next?

This becomes valuable when paired with durable storage, repos, docs, issue trackers, calendars, and artifacts. Continuity reduces restart cost.

Failure mode: continuity without judgment becomes a longer transcript, not a smarter system.

## 3. Artificial Continuity

Artificial continuity is not pretending the agent has lived experience.

It is the compression of prior work into reusable judgment.

It answers:

> What pattern has already been discovered, tested, named, and promoted?

The goal is not nostalgia. The goal is avoiding rediscovery tax.

## 4. WWDD

WWDD means "What Would Drew Do?"

It is not personality simulation. It is not roleplay.

WWDD is Judgment-as-a-Service.

It answers:

> What has Drew repeatedly done when confronted with this class of problem?

Evidence beats stated preference. Repeated decisions matter more than opinions.

Primary evidence sources:

- Chat exports
- Git commits
- Pull requests
- Tickets
- Docs
- Email
- Calendar events
- Project artifacts
- Repeated interventions

## 5. Agent Relativity

Agent Relativity emerged from the insight that an unanchored agent experiences only requests, while an anchored agent experiences consequences.

An agent connected to repositories, calendars, docs, tickets, emails, and history can perceive operational time through artifacts.

Not consciousness.

Not human memory.

Perspective-like behavior created by intelligence plus evidence trails.

## 6. Artifact-Mined Judgment

The strongest enterprise-agent opportunity is not generic chatbot Q&A.

It is mining operational artifacts for repeated decisions, recurring workarounds, hidden process debt, and tribal knowledge.

Reality leaves fossils.

Tickets, commits, comments, meeting notes, emails, and support cases reveal how work actually happens.

## 7. Portable Judgment

If judgment can be distilled, it can be packaged.

A prompt is fragile.

A doctrine pack is inspectable, versioned, cloneable, and improvable.

Portable Judgment means another agent can inherit operating principles without needing the full original chat history.

## 8. Agent Closet

Agent Closet is the artifact form of Portable Judgment.

It stores orientation packs for agents such as Quartermaster, Prospector, Foreman, and Holocron.

The repo is not the agent.

The repo is the closet where the agent keeps its work clothes, tools, scars, and laminated warnings from prior incidents.

## Doctrine

The artifact comes first.

The repo proves the theory by existing.

The manifesto comes later. The merge request comes first.
