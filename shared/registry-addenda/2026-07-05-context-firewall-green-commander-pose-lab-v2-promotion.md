## Promoted discoveries

### 1. Context Firewall Doctrine

Observation: Agent Closet PR #2 names the failure mode that kept reappearing across visual tooling, fiction generation, and Home Center: old project assumptions silently redefine the new target.

Why it matters: false-green work is often not wrong because the implementation is weak. It is wrong because the agent has started proving a substituted artifact.

Reusable rule:

```text
Old context may warn.
It must not silently define the new target.
The current visible noun controls.
```

Evidence: Agent Closet PR #2 adds `shared/doctrine/context-firewall-doctrine.md`, links it through the doctrine index, and records exact-noun preservation, hard interrupts, and proxy-artifact rejection.

Confidence: high as operating doctrine; draft/source-ref boundary remains until PR #2 merges.

Owning clusters: AI Orchestration, Project Archaeology, Animation and Pose Systems, Software Delivery.

### 2. Output Surface Must Match User Intent

Observation: Home Center issue bundle #24-#33 repeats one failure with variations: the user asked for prose, but the system searched, saved, updated, stalled, reported, or produced premise-breaking output instead of the requested visible creative artifact.

Why it matters: a voice-first household assistant steals agency when hidden internal work replaces the visible/listenable answer.

Reusable rule:

```text
A creative request is not complete until the requested creative artifact reaches the user.
Context reads may support the task; they cannot replace the answer.
```

Evidence: issue #28 preserves the strongest premise-custody failure: Lexen was moved from tank commander inside the floating tank to infantry outside it. Issue #33 preserves the active-task stall after a remembered Lexen doctrine document was available. Home Center commits route pure creative requests to dedicated drafting, block tool plans after context read, and make creative drafting timeout-safe.

Confidence: high for Home Center; medium-high reusable assistant doctrine.

Owning clusters: AI Orchestration, Accessibility Engineering, Software Delivery.

### 3. Premise Custody For Creative Generation

Observation: the creative failure was not merely style or quality. It violated explicit role, vehicle, location, and historical constraints.

Why it matters: polished prose can be worse than no prose when it overwrites canon and makes the user fight the machine for custody of their own premise.

Reusable rule:

```text
Preserve named role, point of view, commanded vehicle, location, and historical constraint before optimizing prose.
```

Evidence: Home Center validation prompt now rejects moving a tank commander out of the commanded vehicle, replacing vehicle relationships, or turning grounded historical constraints into symbolism.

Confidence: high for current Home Center bug class; medium reusable writing-system rule.

Owning clusters: Writing Systems, AI Orchestration, Project Archaeology.

### 4. Scene Manifest As Observation Queue

Observation: the Tanks manifest frames scenes as simulation targets: one scene, one observed question, one continuity function.

Why it matters: it prevents chapter-list sludge and forces fiction work to preserve the observation that made the scene useful.

Reusable rule:

```text
Do not ask only what chapter comes next.
Ask what moment the world must observe next.
```

Evidence: `docs/continuity/alpha-is-lexen-vigil.md` now names Tanks as the historical root system and lists phase-scoped observation questions from home-before-war through Normandy, Cold War/Paradise, analog future, and Infinite Brutality lineage.

Confidence: high for the Tanks/Holding Vigil/Infinite Brutality continuity bridge; medium-high reusable writing/game design pattern.

Owning clusters: Writing Systems, Project Archaeology, Game Development.

### 5. Green Authority Becoming Humane Authority

Observation: the Lexen green-commander material corrected the old read from recruit-greenness to command-greenness: rank outruns the soul, and authority is learned through conduct under witness.

Why it matters: this gives future scenes a command arc instead of generic coming-of-age military bravado.

Reusable rule:

```text
Authority is not proven by taking damage.
Humane command is proven by knowing when ritual must stop.
```

Evidence: Drive Lexen doctrine and simulation materials preserve the cigarette as command-language, not adolescent rebellion, and Tango as a model of reverence, limits, and small decencies.

Confidence: medium-high as writing doctrine; Drive remains canonical until imported into repo scene cards.

Owning clusters: Writing Systems, Game Development.

### 6. Pose Lab V2 Clean Surface Extraction

Observation: Pose Lab V2 is not a continuation of the failed Pose Lab proof maze. It is a smaller salvage surface with real Meshy character and real sabre assets, direct manipulation, explicit smoke tests, debug proof panels, and hosted review routes.

Why it matters: salvage succeeds only when the new surface is stricter about evidence than the failed system, not when it imports old gates and calls them maturity.

Reusable rule:

```text
A salvage surface should be smaller than the failed system,
but its evidence lane must be stricter, not weaker.
```

Evidence: Pose Lab V2 smoke tests require real Meshy GLB, real sabre GLB, dynamic cache-busted ready-pose loading, explicit RightHand-owned SaberGrip FK, shared Saber/Pose manipulation, and no runtime animation retargeting.

Confidence: high for Pose Lab V2 implementation boundary; medium reusable salvage doctrine until repeated in another project.

Owning clusters: Animation and Pose Systems, Game Development, Project Archaeology, Software Delivery.

### 7. FPV Camera As Perceptual Review Tool

Observation: FPV mode changes the review question from outside attachment to first-person usability. The later escape-button fix proves that perception review still needs a visible return path.

Why it matters: review modes can become traps if they hide the control needed to leave them. First-person evidence is only useful if the review surface remains operable.

Reusable rule:

```text
Perceptual review must preserve an escape hatch.
A view mode that proves experience must not trap the reviewer inside it.
```

Evidence: Pose Lab V2 added URL boot into `view=fpv`, head-forward FPV sync, a mobile control dock, and an explicit FPV escape selector that remains visible while other edit chrome is hidden.

Confidence: high for Pose Lab V2 UI/visual review; medium reusable UI doctrine.

Owning clusters: Animation and Pose Systems, Accessibility Engineering, Game Development.

### 8. Orientation-Aware Visible Delta Proof

Observation: earlier ready-pose proof could be position-only. Pose Lab V2 now records quaternion angle, hand-axis screen delta, blade-axis angle, and blade-axis screen delta.

Why it matters: wrist rotation and blade orientation can be visually important even when hand origin barely moves.

Reusable rule:

```text
A visible pose proof must measure orientation when orientation is what the viewer notices.
```

Evidence: `debugReadyDelta` now reports orientation deltas and has explicit `VISIBLE_ORIENTATION_DELTA` / `NO_VISIBLE_ORIENTATION_DELTA` states.

Confidence: high for pose/weapon work; medium-high reusable visual QA rule.

Owning clusters: Animation and Pose Systems, Software Delivery.

### 9. Timeout Budget Must Match Hosted Function Window

Observation: Home Center creative drafting gained an explicit 75-second provider timeout below the Firebase function ceiling, then turns timeout, provider failure, empty output, and malformed output into visible failure states with task completion cleanup.

Why it matters: a cloud function that waits until platform timeout does not fail usefully. The user needs a bounded, named failure with state cleanup.

Reusable rule:

```text
Set provider timeouts inside the product boundary, not at the platform cliff.
On failure, complete or clean up task state so stale active tasks do not poison the next turn.
```

Evidence: `CREATIVE_OPENAI_TIMEOUT_MS = 75000`, `AbortController`, `creative_provider_timeout`, and `attachCreativeTaskComplete` are now encoded in Home Center implementation and regression checks.

Confidence: high for Home Center; high reusable Software Delivery doctrine.

Owning clusters: Software Delivery, AI Orchestration, Accessibility Engineering.
