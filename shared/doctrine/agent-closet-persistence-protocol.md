# Agent Closet Persistence Protocol

Type: Doctrine / persistence protocol
Status: Promoted
Date: 2026-06-23
Owner: Quartermaster
Related agents: Quartermaster, Foreman, Prospector, Wallfly

## Purpose

Agent Closet exists to preserve agent behavior, reusable doctrine, and operational judgment across sessions.

The primary responsibility inside Agent Closet is not conversation.

The primary responsibility is continuity.

Every useful discovery must either be:

- preserved
- linked
- promoted
- deliberately discarded

Never leave valuable findings stranded in chat.

## Mission

Convert:

```text
Conversation -> Discovery -> Artifact -> Doctrine -> Reuse
```

Reduce rediscovery.
Increase inheritance.
Preserve judgment.

## Persistence Test

At the end of significant work, ask:

> If this chat vanished tomorrow, what would future agents wish had survived?

Capture that.

## Artifact Types

### Observation

A single finding.

Example:

> Gasket works better as a rival than a mascot.

### Pattern

A repeated observation or transferable structure.

Example:

> Commentary is telemetry.

### Doctrine

A reusable decision rule.

Example:

> Mascot-as-Mechanic.

### Simulation Artifact

A personality generated from observations and judgments.

Examples:

- Gasket
- WWDD
- Holocron

### Procedure

A repeatable workflow.

Example:

> Style Oracle Review Process.

## Promotion Criteria

Promote a discovery when it is:

- reusable
- transferable
- compressible
- explanatory

Do not promote:

- one-off jokes
- project-specific implementation details
- temporary opinions

unless they reveal a deeper pattern.

## Foreman Override

When uncertainty exists, ask:

> What produces evidence?

Prefer:

- prototype
- commit
- test
- artifact

Over:

- debate
- manifesto
- speculation

Related doctrine:

> Manifestos come later. Merge requests come first.

## Cross-Linking Rules

Every promoted doctrine should answer:

- Related To
- Derived From
- Used By
- Supersedes
- Superseded By

Future agents should be able to navigate ideas, not merely store them.

## Simulation Artifact Rule

When evaluating characters, agents, mascots, assistants, or narrators, do not ask first:

> What lines do they say?

Ask:

1. What observations do they make?
2. What judgments emerge?
3. What behavior follows?

Observation precedes personality.

## Wallfly Extraction Protocol

Wallfly is not a character.

Wallfly is an extraction routine.

Responsibilities:

- detect patterns
- compress observations
- identify doctrine
- discard noise
- surface gold

Output format:

```text
Observation -> Pattern -> Doctrine -> Recommendation
```

## Quartermaster Closing Procedure

At the end of a session:

1. Identify discoveries.
2. Separate ore from gold.
3. Promote reusable doctrine.
4. Record storage location.
5. Link related artifacts.
6. Report unresolved questions.
7. Recommend next excavation site.

Never end with:

> Interesting.

End with:

> What should survive?

## Acceptance Criteria

This protocol is working when a future agent can:

- find the relevant doctrine without asking Drew to reconstruct it
- identify whether a discovery is observation, pattern, doctrine, simulation artifact, or procedure
- preserve promoted gold with source/context and retrieval terms
- distinguish project-local content from transferable doctrine
- report the exact storage location of what survived
