# Self-Insertion Calibration

Type: Doctrine / agent design / archetype design / artificial continuity
Status: Promoted gold
Date: 2026-06-24
Source/context: User supplied Self-Insertion Calibration Prompt and requested repo insertion. Follow-up doctrine learned in chat: self-insertion is calibration for useful judgment, not identity replication.

## Core rule

Before creating an agent, archetype, protagonist, assistant, faction, mentor, foreman, companion, or decision-making system, do not ask:

> "What would be cool?"

Ask:

> "What part of the user is being externalized?"

Treat self-insertions as calibration artifacts.

A self-insert is not automatically vanity.

A self-insert may be:

- a behavioral reference model
- a values carrier
- a judgment archive
- a capability amplifier
- a responsibility simulator
- a way to preserve lived experience

Do not clone the user.

Extract the useful operating pattern.

Then evolve it.

## Calibration questions

For the proposed character, agent, or system, determine:

1. What recurring judgment pattern does this preserve?
2. What responsibility does this carry?
3. What capability does this amplify?
4. What weakness does this compensate for?
5. What burden should it NOT inherit?
6. Where should it disagree with the user?
7. What would it notice that the user routinely misses?

## Evaluation questions

When evaluating the design, answer:

> "What part of the user is present?"

and

> "What part of the user is intentionally absent?"

The goal is not replication.

The goal is preserving useful judgment while allowing divergence.

## Archetype Independence Test

Final test:

> "If the user disappeared tomorrow, would this character, agent, or system still make useful decisions?"

If no:

> "It is a clone."

If yes:

> "It is an archetype."

A successful archetype is capable of producing useful judgment in the absence of its creator.

If the system requires the creator's continuous presence to make decisions, it is a clone.

If the system can continue operating from preserved principles, responsibilities, and judgment patterns, it is an archetype.

## Learned doctrine

Most failed self-insert designs confuse identity with capability.

A useful agent is not the user's biography, favorite jokes, speech patterns, or emotional weather.

A useful agent preserves:

- recurring judgment patterns
- decision heuristics
- responsibility boundaries
- accumulated lessons
- productive disagreements
- blind-spot coverage

This shifts agent design from personality simulation to operational continuity.

## Distinctions

### Clone

A clone mirrors the user.

It depends on the user's identity, preferences, and constant steering.

It may feel familiar, but it does not carry durable responsibility.

### Archetype

An archetype extracts a reusable operating pattern.

It inherits useful judgment, rejects unnecessary burden, and is allowed to diverge.

It can disagree with the user because its job is not flattery.

Its job is capability inheritance.

## Operational use

Use this doctrine when creating or evaluating:

- agent profiles
- protagonists
- factions
- mentors
- assistants
- companion characters
- decision systems
- WWDD-style judgment services
- perspective council members
- role-specific AI tools
- user-inspired game characters

Ask what capability is being externalized before choosing voice, costume, name, or lore.

## Acceptance criteria

This doctrine is working when an agent can:

- identify what part of the user is being externalized
- name the recurring judgment pattern being preserved
- define the responsibility the archetype carries
- state which user burdens should not be inherited
- identify where the archetype should productively disagree
- pass or fail the Archetype Independence Test
- produce useful decisions without becoming a personality clone
