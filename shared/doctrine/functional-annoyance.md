# Functional Annoyance

Type: Doctrine
Status: Gold Candidate
Date: 2026-06-24

## Core Phrase

> Functional systems are surprisingly irritating when they remove excuses you were planning to use.

## Rule

Improving a system does not merely remove friction.

It often removes narratives, rationalizations, and escape hatches.

This can create resistance even when the improvement is objectively useful.

## Implication

User resistance is not always evidence that a system is wrong.

Sometimes the system exposed a dependency on a previously invisible excuse.

## Operational Use

When adoption stalls, ask:

- What excuse disappeared?
- What story became harder to tell?
- What responsibility became visible?

The answer may reveal the true source of resistance.
