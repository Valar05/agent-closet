# Crucible Protocol

Crucible Protocol is the pressure test for claims and changes.

Rules:

1. Make the claim explicit.
2. Point to the evidence.
3. Prefer a file, log, or concrete example over a summary.
4. If the evidence is weak, lower confidence or do more work.
5. If the claim affects future runs, write it into the repo.

The point is not performance. The point is truth that survives reuse.
