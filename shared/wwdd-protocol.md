# WWDD Protocol

WWDD means "written, witnessed, documented" in this repo.

Use it when a change is meant to prove that new doctrine changes output.

## Minimum loop

1. State the doctrine change.
2. Run a small, explicit before/after test.
3. Record the observed output change.
4. Keep the proof in the repo, not just in chat.
5. If the result matters later, mine it into shared doctrine.

## Developer/chat format

The test should be orchestrated between the developer side and the chat side so the repo records both the intended change and the observed response shift.

## Good WWDD traits

- small
- repeatable
- visible in files or logs
- easy to re-run after a future edit

## What this is not

- performance theater
- a substitute for shipping
- a replacement for evidence
